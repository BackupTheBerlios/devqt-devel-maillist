How I got interested in this project: I wanted an IDE with Qt integration.
Eclipse is very good for C++ but it doesn't provide Qt support. I've seen
this project on QtCentre and though why not contribute... I don't care if I
use an IDE made with Qt or Java or whatever what I want it is to do the job.
Since you want to do this in Qt from scratch I though it may be a good time
for some contribution :) (although I don't have Qt experience I can adapt
quickly)

I've attached a small design document about what I'd like to have in an IDE.

Yes, a roadmap and good planning is needed. I am happy that Thiago Santos
has Qt experience; it is a good sign :)
We talked on IRC about starting DevQt with the interface and the core of the
IDE .. this means no integration with any tools, just the basic interface
with configuration system, a basic editor, the plugin system and the build
system for the IDE itself(the program should build on every platform that Qt
builds). Thiago offered to write the UI system. What do you think about this
?

Another question that I have is about the Qt trademark. I am not sure if
DevQt is a safe name; Trolltech may not be very happy about this. Maybe DevQ
would be a better name ?
